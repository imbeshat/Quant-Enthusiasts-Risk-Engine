LOCAL DEVELOPMENT (with hot-reload):
====================================
1. Fix line endings:
   dos2unix start.sh

2. Build Docker image:
   docker build -t quant-engine:latest .

3. Run in development mode:
   docker run -d \
     -p 8080:5000 \
     -p 3000:3000 \
     -e DEV_MODE=true \
     --name quant-dev \
     quant-engine:latest

4. Access:
   - React Dev Server: http://localhost:3000
   - Flask API: http://localhost:8080
   - Health Check: http://localhost:8080/health

5. View logs:
   docker logs -f quant-dev

6. Stop:
   docker stop quant-dev && docker rm quant-dev


LOCAL PRODUCTION (testing production build):
============================================
1. Build:
   docker build -t quant-engine:latest .

2. Run:
   docker run -d \
     -p 8080:5000 \
     --name quant-prod \
     quant-engine:latest

3. Access:
   - Application: http://localhost:8080
   - Health Check: http://localhost:8080/health

4. View logs:
   docker logs -f quant-prod


RENDER.COM DEPLOYMENT:
=====================
1. Push code to GitHub repository

2. Create new Web Service on Render:
   - Connect your GitHub repository
   - Render will auto-detect render.yaml

3. Environment variables (auto-set by render.yaml):
   ✓ PYTHON_VERSION=3.11.0
   ✓ NODE_VERSION=18
   ✓ RENDER=true
   ✓ PORT=10000

4. Deploy:
   - Render will automatically build and deploy
   - Build command: bash start.sh
   - Start command: cd python_api && python app.py

5. Access:
   https://quant-enthusiasts-risk-engine.onrender.com

6. Monitor:
   - Logs: Render Dashboard > Logs
   - Metrics: Render Dashboard > Metrics

TROUBLESHOOTING:
===============
Issue: "start.sh: command not found"
Fix: Run dos2unix start.sh

Issue: React files not found
Fix: Check that dist/ directory exists in Docker image
     docker exec quant-prod ls -la /app/react_dashboard/dist

Issue: CORS errors
Fix: Verify CORS origins in app.py match your domain

Issue: API timeout
Fix: Check Render logs for errors
     Verify PORT environment variable is 10000

Issue: 404 on routes
Fix: Ensure Flask catch-all route serves index.html
     Check that static_folder points to correct directory
